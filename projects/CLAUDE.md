# Projects — Per-Project Workspaces

Each subfolder is a project workspace. Projects hold everything **bound to a specific customer, environment, or deliverable** — extracted artifacts, investigation findings, test execution results, and optionally a GH repo with deployable code.

## Sharing Rules

**Everything in `projects/` is personal** — never reaches the shared team repo. Project CLAUDE.md files and test-assets.md are tracked by the harness for context.

## Project Structure

```
projects/{name}/
  CLAUDE.md              # Project context (environment, write policy, scope)
  test-assets.md         # Platform-side test components (forms, WS, dashboards)
  environment.json       # Platform version profile (generated by env:profile)
  repo/                  # Independent GH repo (deployable code, gitignored)
  extracts/              # Extracted data from VV admin (gitignored)
  analysis/              # Project-specific research & findings
  tools/                 # Project-specific operational scripts (not in repo)
  testing/               # Test execution output (gitignored)
    {task}/
      status.md          # Task rollup: per-component summary
      {component}/
        status.md        # Per-slot pass/fail with actual values
        runs/            # Immutable execution records
        results.md       # Session log (optional)
```

Not every project needs every subfolder. They're created as needed.

## Shared vs. Project-Specific

Tools, research, and documentation can be project-specific or shared:
- **In `repo/`**: code that ships with the project (project-specific scripts, tools)
- **In `tools/`** (workspace): operational scripts for working with the project (not deployed)
- **In `analysis/`**: investigation findings scoped to this project

**Promotion:** When a second project needs something, promote it to the harness level (`tools/`, `research/`, `docs/`).

## Creating a New Project

Use `/@-create-project <name>` — supports VV and non-VV projects, optional `--repo` flag.

Manual steps:
1. Create `projects/{name}/` with CLAUDE.md
2. If VV: configure `.env.json` under `servers.{name}.customers.{name}`
3. If repo: `git clone <url> projects/{name}/repo/` or `mkdir repo && cd repo && git init`
4. Run `npm run env:profile -- --project {name}` (VV only)
