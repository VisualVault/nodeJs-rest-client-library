# Form Templates — XML Export Analysis & Generation

VV form template XML exports used for understanding the export format, analyzing field structures, and generating improved templates.

## Files

| File                      | Description                                                                                                                                                                                                   |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `datetest-original.xml`   | Original DateTest form template export (26 calendar fields, 5 WS fields, 2 buttons). Field names are `DataField1`–`DataField28`. No labels.                                                                   |
| `subscription-pack.xml`   | Production Subscription Pack form export. Reference for field types not in DateTest: FieldLabel, DropDownList, Checkbox, FormIDStamp, UploadButton. Also shows Groups/Conditions and rich scripting patterns. |
| `datetest-v2.xml`         | Redesigned DateTest template with descriptive field names, labels, and grouped layout. Generated by `generate-datetest-v2.js`.                                                                                |
| `generate-datetest-v2.js` | Node.js generator script. Produces `datetest-v2.xml` with exact structural fidelity to the original VV XML format.                                                                                            |

## DateTest v2 Design

### Field naming

| Prefix | Group                    | Example          |
| ------ | ------------------------ | ---------------- |
| `Cal_` | Base — no initial value  | `Cal_A`, `Cal_H` |
| `Pre_` | Preset date (03/01/2026) | `Pre_A`, `Pre_H` |
| `Cur_` | Current date (auto-fill) | `Cur_A`, `Cur_H` |

### Config key

| Letter | enableTime | ignoreTZ | useLegacy | Description                   |
| :----: | :--------: | :------: | :-------: | ----------------------------- |
|   A    |    OFF     |   OFF    |    OFF    | Date-only                     |
|   B    |    OFF     |    ON    |    OFF    | Date-only + IgnoreTZ          |
|   C    |     ON     |   OFF    |    OFF    | DateTime                      |
|   D    |     ON     |    ON    |    OFF    | DateTime + IgnoreTZ           |
|   E    |    OFF     |   OFF    |    ON     | Date-only + Legacy            |
|   F    |    OFF     |    ON    |    ON     | Date-only + IgnoreTZ + Legacy |
|   G    |     ON     |   OFF    |    ON     | DateTime + Legacy             |
|   H    |     ON     |    ON    |    ON     | DateTime + IgnoreTZ + Legacy  |

### Layout

2-column grid: Modern (A–D) left, Legacy (E–H) right. Three sections stacked vertically (Base, Preset, Current Date), each with labels describing every field's config. WS Harness section at bottom with 6 text fields (WSAction, WSConfigs, WSRecordID, WSInputDate, WSInputFormats, WSResult).

### Regenerating

```bash
node tasks/form-templates/generate-datetest-v2.js > tasks/form-templates/datetest-v2.xml
```
